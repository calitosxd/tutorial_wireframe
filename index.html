3 Herramientas Open Source
para Wireframes

Evaluamos las principales alternativas gratuitas para crear wireframes en proyectos de análisis y diseño de sistemas, comparando características, ventajas y casos de uso.

🎨
Penpot
// Web · Cloud · Open Source

La alternativa open source más completa a Figma. Soporta colaboración en tiempo real, componentes reutilizables y prototipos interactivos, todo desde el navegador.

Colaboración RT Prototipos Componentes MPL-2.0

✓ Pros

  • Colaboración en tiempo real
  • Prototipos interactivos
  • Sin instalación
  • Exporta PNG, SVG, CSS
  • Comunidad activa

× Contras

  • Requiere internet
  • Docker p/ self-host
  • Curva vs Figma
Visitar penpot.app ↗
✏️
Pencil Project
// Escritorio · Offline · GPL-2.0

Aplicación de escritorio especializada en wireframes con librerías UI predefinidas para web, Android e iOS. Ideal para uso individual sin conexión.

Offline Librerías UI GPL-2.0

✓ Pros

  • Funciona offline
  • Librería UI completa
  • Fácil para principiantes
  • Exporta PDF/HTML

× Contras

  • Sin colaboración RT
  • Sin actualizaciones
  • No prototipos
  • UI desactualizada
Visitar pencil.evolus.vn ↗
🖊️
Excalidraw
// Web · PWA · MIT License

Pizarra virtual open source para bocetos y wireframes con estilo visual de "dibujado a mano". Perfecta para ideas rápidas y reuniones de levantamiento de requerimientos.

Bocetos rápidos Colaboración MIT

✓ Pros

  • Sin registro obligatorio
  • Muy fácil de usar
  • Colaboración por enlace
  • PWA offline

× Contras

  • Sin librería UI
  • Solo baja fidelidad
  • Sin prototipos

Tutorial Paso a Paso

Cómo usar Penpot para
crear Wireframes

20 pasos organizados en 8 módulos para crear wireframes profesionales de la Tienda La Esquinita usando Penpot desde cero.

01 Registro y Acceso
PASO 01
Acceder al sitio web

Abrir Chrome o Firefox y visitar https://penpot.app. Hacer clic en "Start for free" o "Get started".

No se requiere tarjeta de crédito. El acceso es completamente gratuito.
PASO 02
Crear cuenta y explorar el Dashboard

Registrarse con correo o Google. En el Dashboard hacer clic en "New project" y nombrarlo: Wireframe Tienda La Esquinita.

02 Crear el Archivo de Diseño
PASO 03
Crear un nuevo archivo

Dentro del proyecto, hacer clic en "+" o "New file". Se abrirá el editor de diseño en una nueva pestaña.

PASO 04
Conocer la interfaz del editor

El editor tiene 3 áreas: Panel izquierdo (capas y activos), Canvas central (lienzo), y Panel derecho (propiedades del elemento).

03 Crear Frames (Pantallas)
PASO 05
Insertar un Frame

En la barra de herramientas superior, hacer clic en el ícono Frame (tecla F). Seleccionar "Android — Small" (360×640 px) para simular el celular de Doña María.

Atajo: presiona F + arrastra en el canvas para dibujar el frame directamente.
PASO 06
Nombrar el Frame

En el panel de capas (izquierda), doble clic sobre el nombre del frame y renombrarlo: "Pantalla Principal — Inventario".

04 Agregar Elementos UI
PASO 07
Barra de navegación superior

Herramienta Rectángulo (tecla R). Dibujar 360×56px. Color de relleno: #1a3a5c. Agregar texto (tecla T) con "La Esquinita".

PASO 08
Tarjetas de productos

Rectángulo 320×80px, bordes redondeados (radio 8px), fondo blanco. Agregar nombre del producto, cantidad disponible e indicador de estado (verde/rojo según stock).

Duplicar con Ctrl+D para crear la lista de productos rápidamente.
PASO 09
Botón de alerta de stock

Rectángulo 160×44px, relleno #e74c3c, texto blanco "⚠ Stock Bajo". Posicionarlo en la parte inferior del frame como acción rápida.

PASO 10
Barra de navegación inferior

Rectángulo 360×56px, fondo #f5f5f5. Agregar 4 íconos: Inicio, Ventas, Crédito y Reportes.

05 Componentes Reutilizables
PASO 11
Crear un componente

Seleccionar la tarjeta de producto. Clic derecho → "Create component" (Ctrl+K). El componente aparece en Assets. Al modificarlo, todos los usos se actualizan.

Los componentes son la clave para diseños consistentes y escalables.
PASO 12
Reutilizar en otras pantallas

Crear frame de "Gestión de Créditos". Desde el panel Assets, arrastrar el componente al nuevo frame. Los cambios en el original se propagan automáticamente.

06 Prototipo Interactivo
PASO 13
Activar modo Prototype

En el panel derecho, hacer clic en la pestaña "Prototype" (ícono de triángulo/flecha). Se activará el modo de conexión entre pantallas.

PASO 14
Conectar pantallas

Seleccionar el botón "Ventas". Aparece un punto de conexión azul. Arrastrar hacia el frame de la pantalla de Ventas. Esto crea un flujo de navegación interactivo.

PASO 15
Previsualizar el prototipo

Clic en "▶ Play" en la barra superior (Ctrl+Alt+P). Se abre previsualización donde se navega entre pantallas como si fuera la app real.

Compartir el enlace del prototipo con Doña María para obtener feedback antes de desarrollar.
PASO 16
Agregar transiciones

En la conexión entre frames, configurar tipo de transición: Slide, Fade o Push. Ajustar duración (ms) y curva de animación para simular la experiencia móvil.

07 Colaboración en Tiempo Real
PASO 17
Invitar a un colaborador

En el Dashboard, abrir el proyecto → "Share". Ingresar el correo del compañero. Rol "Can edit" para editar o "Can view" solo para revisar.

PASO 18
Edición simultánea

Con varios usuarios en el archivo, se ven los cursores de cada uno en tiempo real identificados con nombre y color. Los cambios se reflejan instantáneamente.

Ideal para que el equipo revise el wireframe con el docente sin enviar archivos.
08 Exportar el Wireframe
PASO 19
Exportar como imagen

Seleccionar el frame. En el panel derecho → sección "Export" → "+" para agregar formato. Elegir PNG a 2x de resolución para mayor calidad en documentos.

PASO 20
Incluir en la ERS

Exportar cada pantalla como PNG. Incluir en la sección Anexos del documento ERS, referenciando cada wireframe con su RF (ej. RF-01, RF-03, RF-10).

El vínculo wireframe → RF demuestra trazabilidad completa según ISO 29148:2018.

Trazabilidad

Relación Wireframes ↔ ERS

Cada pantalla del wireframe se relaciona directamente con un Requerimiento Funcional (RF) del documento ERS de la Tienda La Esquinita, garantizando trazabilidad completa según ISO/IEC/IEEE 29148:2018.

RF-01
Pantalla Principal — Inventario
Mostrar en tiempo real la cantidad disponible de cada producto. Incluye indicadores visuales de stock bajo.
RF-02
Alerta de Stock Bajo
Generar alerta automática cuando el stock llegue al umbral mínimo configurado (por defecto 5 unidades).
RF-03
Registro de Venta
Registrar venta asociando productos, cantidades y medio de pago (efectivo, Nequi o Daviplata).
RF-06, RF-07, RF-08
Gestión de Créditos
Registrar deudas, bloquear crédito por límite ($50.000) y mostrar lista de clientes con saldo pendiente.
RF-10, RF-11
Catálogo PWA — Clientes
Catálogo de productos accesible por clientes vía PWA y opción de apartar productos sin registrarse.
RF-05
Reporte Diario de Ventas
Reporte generado al cierre del día desglosado por medio de pago: efectivo, Nequi y Daviplata.
RF-15
Pedido a Proveedor
Registrar pedidos anticipados a proveedores con lista de productos y cantidades requeridas.
RF-17
Indicador Modo Offline
Visualización del estado de sincronización: online, offline (operaciones en cola) o sincronizando.
Conclusiones

¿Por qué elegir Penpot?

Las tres herramientas analizadas ofrecen alternativas válidas según el contexto. Penpot destaca como la opción más completa para proyectos académicos y profesionales.

🤝
Colaboración sin límites

Penpot permite que todo el equipo trabaje simultáneamente en el mismo archivo, ideal para proyectos grupales y revisiones con el docente en tiempo real.

⚡
Potencia profesional gratis

Prototipos interactivos, componentes reutilizables y exportación CSS son funciones que herramientas de pago cobran a precios elevados. Penpot las ofrece sin costo.

🔗
Integración con la ERS

Cada pantalla creada en Penpot puede vincularse directamente con un RF de la ERS, garantizando trazabilidad completa según ISO/IEC/IEEE 29148:2018.
